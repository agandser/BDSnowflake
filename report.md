# Отчёт по лабораторной работе №1

Выполнил: Агафонов Андрей Сергеевич, группа М8О-315Б-23.

## Тема работы

Нормализация исходных данных в схему "снежинка" на основе набора CSV-файлов с данными о покупателях, продавцах, товарах, магазинах и поставщиках.

## Цель работы

Нужно было загрузить исходные данные в PostgreSQL, выделить из них основные сущности и построить аналитическую модель данных, удобную для дальнейших запросов и проверки.

## Что было сделано

Для выполнения работы я использовал `docker-compose.yml`, чтобы база данных поднималась одной командой. При старте контейнера автоматически выполняются SQL-скрипты из папки `sql`: сначала создаётся сырьевая таблица, затем в неё загружаются все 10 CSV-файлов, после этого создаются таблицы схемы "снежинка" и данные раскладываются по измерениям и фактам.

В качестве центральной таблицы была выбрана `fact_sale`, потому что в исходных данных основной бизнес-событие - это продажа товара. Вокруг неё вынесены измерения `dim_customer`, `dim_seller`, `dim_store`, `dim_supplier`, `dim_product`, `dim_product_category`, `dim_pet` и `dim_location`. Отдельно оставлена таблица `raw_mock_data`, чтобы можно было проверить загрузку исходного слоя.

Для локаций использована одна таблица `dim_location`, но с полем `location_role`, чтобы одинаковые по названию города или страны записи из разных сущностей не смешивались между собой при загрузке покупателей, продавцов, магазинов и поставщиков.

## Использованные скрипты

1. `sql/01_create_raw_table.sql` - создаёт таблицу `raw_mock_data`.
2. `sql/02_load_raw_data.sql` - загружает исходные CSV-файлы.
3. `sql/03_create_snowflake_tables.sql` - создаёт очищающее представление и итоговые таблицы.
4. `sql/04_fill_snowflake_tables.sql` - переносит данные из сырьевого слоя в измерения и факты.
5. `sql/05_checks.sql` - содержит запросы для проверки результата.

## Результат

После запуска контейнера в базе формируется готовая схема "снежинка", а таблица фактов содержит все продажи из исходных файлов. Проверочный скрипт показал такие результаты: `raw_mock_data` - 10000 строк, `fact_sale` - 10000 строк, `dim_customer` - 10000, `dim_seller` - 10000, `dim_store` - 10000, `dim_supplier` - 10000, `dim_product` - 10000, `dim_product_category` - 15, `dim_pet` - 9, `dim_location` - 28083. Для визуальной проверки структуры в репозиторий добавлена схема `docs/lab1-schema.png`.

Дополнительно был выполнен аналитический запрос по категориям питомцев. Максимальная выручка получилась у категории `Birds` - `534985.37`, далее идут `Dogs` - `511599.84`, `Fish` - `510757.04`, `Reptiles` - `491329.75` и `Cats` - `481180.12`.
